Live Your Dreams Design System

Strategie: Wir adaptieren das Porsche Design System (v3) für Live Your Dreams. Alle Porsche-Referenzen werden entfernt und durch unsere CI Guidelines ersetzt. Das System läuft unter designsystem.liveyourdreams.de (intern, nicht öffentlich).

Warum Porsche Design System als Basis?

  • Enterprise-Grade: Battle-tested, barrierefrei (WCAG 2.1 AA), performant
  • Vollständig: 50+ Komponenten, Dark Mode, Responsive
  • Tech-Stack kompatibel: React, Web Components, Design Tokens
  • Dokumentation: Storybook, Guidelines, Best Practices

5. Nginx Config mit Security Headers

# docker/nginx.conf
server {
    listen 80;
    server_name designsystem.liveyourdreams.de;

    # Basic Auth - PFLICHT!
    auth_basic "LYD Design System - Internal Only";
    auth_basic_user_file /etc/nginx/.htpasswd;

    # Security Headers - KRITISCH!
    add_header X-Frame-Options "DENY" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Robots-Tag "noindex, nofollow, noarchive, nosnippet" always;
    add_header Referrer-Policy "no-referrer" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';" always;

    # IP Allowlist (optional aber empfohlen)
    # allow 10.0.0.0/8;  # VPN range
    # allow 192.168.1.0/24;  # Office network
    # deny all;

    location / {
        proxy_pass http://design-system:6006;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /playground {
        proxy_pass http://design-system:3000;
    }
    
    # Block search engine crawlers explicitly
    location = /robots.txt {
        return 200 "User-agent: *\nDisallow: /\n";
    }
}
# Generate .htpasswd with strong password
htpasswd -c -B docker/.htpasswd lyd_team
# Enter strong password when prompted

6. CI Gate - Porsche Remnants Check

#!/bin/bash
# scripts/check-porsche-remnants.sh

echo "🔍 Checking for Porsche remnants..."

FOUND=0

# Check for text references
if grep -r "porsche\|Porsche\|PORSCHE" \
  --exclude-dir=node_modules \
  --exclude-dir=.git \
  --exclude="*.map" \
  packages/ apps/ 2>/dev/null; then
  echo "❌ Found 'porsche' text references"
  FOUND=1
fi

# Check for pds- prefixes
if grep -r "pds-\|\.pds-\|'pds-\|\"pds-" \
  --exclude-dir=node_modules \
  --exclude-dir=.git \
  packages/ apps/ 2>/dev/null; then
  echo "❌ Found 'pds-' prefixes"
  FOUND=1
fi

# Check for @porsche-design-system
if grep -r "@porsche-design-system" \
  --exclude-dir=node_modules \
  --exclude-dir=.git \
  . 2>/dev/null; then
  echo "❌ Found '@porsche-design-system' imports"
  FOUND=1
fi

# Check for Porsche assets
if find packages/assets apps/assets -type f \
  \( -name "*porsche*" -o -name "*pds*" \) 2>/dev/null | grep -q .; then
  echo "❌ Found Porsche-named assets"
  FOUND=1
fi

if [ $FOUND -eq 0 ]; then
  echo "✅ No Porsche remnants found"
  exit 0
else
  echo "❌ Porsche remnants detected - must be removed!"
  exit 1
fi
# Add to package.json scripts
"scripts": {
  "check:porsche": "./scripts/check-porsche-remnants.sh",
  "prebuild": "npm run check:porsche",
  "precommit": "npm run check:porsche"
}

Porsche → LYD Migration

Search & Replace Matrix

Porsche Term LYD Replacement Files
porsche lyd *.ts, *.tsx, *.js, *.json
Porsche Live Your Dreams *.md, comments
pds lds CSS classes, prefixes
@porsche-design-system @liveyourdreams/design-system package.json, imports
#E00000 #3366CC Brand primary color

Migration Script

#!/bin/bash
# scripts/migrate-to-lyd.sh

echo "🔄 Starting LYD Migration..."

# Backup first
cp -r packages packages.backup

# Global replacements
find . -type f \( -name "*.ts" -o -name "*.tsx" -o -name "*.js" -o -name "*.jsx" \) \
  -not -path "./node_modules/*" \
  -exec sed -i '' 's/porsche/lyd/g' {} \;

find . -type f \( -name "*.ts" -o -name "*.tsx" -o -name "*.js" -o -name "*.jsx" \) \
  -not -path "./node_modules/*" \
  -exec sed -i '' 's/Porsche/Live Your Dreams/g' {} \;

# CSS prefix
find . -type f -name "*.css" -o -name "*.scss" \
  -not -path "./node_modules/*" \
  -exec sed -i '' 's/\.pds-/\.lds-/g' {} \;

# Package names
find . -type f -name "package.json" \
  -exec sed -i '' 's/@porsche-design-system/@liveyourdreams\/design-system/g' {} \;

echo "✅ Migration complete! Review changes and test."

  );

Security & Access Control

WICHTIG: Das Design System ist NICHT öffentlich zugänglich. Nur autorisierte Team-Mitglieder haben Zugriff.

Access Control Layers

  1. DNS: Subdomain nur intern auflösbar (oder Cloudflare Access)
  2. Basic Auth: Nginx-Level Authentication
  3. IP Whitelist: Optional für Production
  4. HTTPS: Let's Encrypt oder eigenes Zertifikat
