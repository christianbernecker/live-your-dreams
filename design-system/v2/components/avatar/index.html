Next.js Integration

// components/Avatar.tsx
interface AvatarProps {
  src?: string;
  alt?: string;
  initials?: string;
  size?: 'xs' | 'sm' | 'md' | 'lg' | 'xl' | 'xxl';
  status?: 'online' | 'away' | 'busy' | 'offline';
  className?: string;
}

export const Avatar = ({ src, alt, initials, size = 'md', status, className = '' }: AvatarProps) => {
  const avatarClasses = \`lyd-avatar \${size} \${className}\`.trim();
  
  return (
    <div className={avatarClasses}>
      {src ? (
        <img src={src} alt={alt || 'Avatar'} />
      ) : initials ? (
        <div className="lyd-avatar-fallback">{initials}</div>
      ) : (
        <UserIcon className="lyd-avatar-icon" />
      )}
      {status && <div className={\`lyd-avatar-status \${status}\`} />}
    </div>
  );
};

API Reference

Class Description
.lyd-avatar Base avatar class with premium styling
.lyd-avatar-wrapper avatar container with consistent spacing
.small Small avatar size (14px - Typography Scale)
.large Large avatar size (18px - Typography Scale)
.primary Primary avatar variant with CI gradient
.success Success state with green styling
.error Error state with red styling and validation
.disabled Disabled state with reduced opacity

Accessibility & Best Practices

Keyboard Navigation

Full keyboard support with intuitive navigation patterns.

  • Tab - Navigate to next element
  • Shift + Tab - Navigate to previous element
  • Enter - Activate element
  • Space - Toggle/select element
  • Escape - Close/cancel action

Screen Reader Support

ARIA labels and semantic HTML ensure compatibility with assistive technologies.

Alt Text

Proper alt text for images and meaningful fallback content for screen readers.

Status Indicators

Status information available to screen readers through ARIA labels and descriptions.

Focus Management

Interactive avatars include proper focus indicators and keyboard navigation support.

Image Optimization

Responsive images with proper loading states and error handling for missing photos.
